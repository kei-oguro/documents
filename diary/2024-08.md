08/17 土
--------

- __園芸__
  - 水やりは11:00にペットボトル4本。台風で一晩中雨だと思っていたのに、水やり不要だったのはあまりなかった。

快晴。

__作り置き__ にモヤシをスパイスで炒めた。モヤシ2袋に塩小さじ足らずだったのに、塩気が強過ぎた。

シャウエッセンのこの袋が脂少な目。多過ぎた袋よりも美味しいが、シャウエッセンらしさが最後の一押しで足りない気がする。他のメーカーの粗挽きソーセージに近く感じる。

__間食__ に、ランチのラーメンの残り汁で充填豆腐を煮たもの。

ココナッツミルクでホットチャイを淹れた。ココナッツミルクと水を1:1にしたが、粘度が高過ぎた。1:2か1:3くらいにすべきかも知れない。

08/16 金
--------

8:00に目覚ましを掛けたが、何度目かのスヌーズで気付かなくて止まったらしい。その後9:00に掛け直して9:30起床。

- __園芸__
  - 水やりは少しだけ。
  - 室外機の上の鉢を、一部は本棚に移動させ、 __青紫蘇__ と __レモン__ は床に移動した。また、 __月桂樹__ と __南高梅__ は鉢を倒した。
  - __月桂樹__ は葉っぱがかなり枯れてる。新しい枝と新芽は元気なんだけど。カレーリーフに日照を遮られて、置く場所が少し可哀相だからフラワースタンドを買ってあげたいんだけど。

また水漏れ。除湿器の運転を弱にしてたから？

`まいばすけっと`に買い出し。モヤシやカットキャベツ、サンドイッチと総菜。

__コーヒー__ をプレスで淹れた。ちょっと濃い目にしたっけ？カルディの豆だけど結構美味しい。

リモート出社。

自宅では作業用BGMを`コブクロ`に。

__ランチ__ は野菜炒めと目玉焼き総菜。総菜はバラした焼き鳥とつくねの盛り合わせで、冷たくても美味しい。

エアコンのパワーを上げたら、水漏れがヒドくなった。エアコンが窓際で、除湿器が部屋の反対側なので、サーキュレーターでエアコン側に空気を送る設定にしてみる。

- __ギャンブル__

  - 微妙に損失の出てるETFもあるけど、折角利益非課税のNISAなのだから利益率の高い順に。
  - __FX__ は円高に賭けるかドル安に賭けるかは微妙だが、岸田や日銀が何円くらいを目指すつもりなのか分からない。
    - ドル円レートの水準で判断するのではなさそう。世論の反発があった時に、その逆にしたがるだけでは。
    - インフレで円安に批判が出たら反応して、そのせいで株価が下がったらまた反応して。
    - そして、岸田はあと2か月かそこら。そのあと植田は、または新首相はどうするのか。

iTunesでシャッフルの解除の仕方が分からなくて困ってたが、あんなところにあったとは。

C#のローカル変数でstructのrefが取れるんだな。でも、コレクションの要素は無理。Arrayには特別に配慮されていて、Span, Memoryはそれぞれ個別に実装されてる。

貸与PCのトラックパッドが効かなくなって焦った。FNキーにトラックパッドのon/offがあった。

お盆期間だから、普段いかなそうなチェーン系でランチを済ませようかと思ったけど、まあまあ遠いコーヒー屋さんの様子を見に行って、その辺で探すことにした。

__ランチ__ は芝公園の`鳥一代`で参鶏湯ラーメン2辛。しっかり辛い。

戻りに`Litコーヒースタンド`でアイスコーヒーをテイクアウト。エチオピア。

暑い中で歩き過ぎたからか、昨日の食事が少ないからか、昼食から戻った後、調子が出ない。

三項演算子の条件にboolベクトルを受けるのは当然じゃないかと思うのだが、hlslとglslではそうなっていない。`Cg`はベクトル。言語から見たら`lerp()`で十分に代わりになるということなんだろうが、HWでより効率的な回路が組み込まれているかも知れず、言語に組み込むことで適切に選択できるのではないか。

視錐台判定に面方程式を使うのは普通だ。面方程式1つなら`dot()`で当然なんだけど、4つ以上あるなら転置しても良いかも知れない。そんなことを思うのは、ローエンドGPUでは`dot()`のスループットが遅いのではないかと疑っているからだ。

明日は台風なのでリモート業務推奨日。自宅に送付してもらった貸与PCが故障したので、別に貸与して頂いているノートPCを持ち帰る。ゲーミングノートで大きい。充電器が大きいのが僕をひるませる。

ノートPCの方を色々セットアップ。

Riderのキーマップをどうするか悩む。

- 田町で飲んで帰る。
  - ランチを食べた `鳥一代`は田町が本店だそうだ。じゃあコッチだけで良かったんだけど。
  - `芝浦ホルモン`本店で __晩酌__ 。
    - 新宿にも店舗があって、アチコチに沢山あるのだと思っていたが、2軒だけらしい？
    - 昭和29年創業を謳っているが、それは焼肉屋の方だそうだ。2Fと3Fが焼肉`はるみ`。今日は焼肉が目当てで来た。ホルモン盛り合わせは、かなり盛りが良い。
    - 次回に串のもつ焼きを食べに来たいと思うくらいに満足した。
    - 後日検索すると、新宿店でも焼肉を食べられるらしい。と思ったら、僕の知ってたのは焼肉の方で西新宿7丁目にあり、別に焼肉の無いもつ焼きの店が歌舞伎町にあるらしい。3軒か。

- __ギャンブル__
  - __FX__ でトルコリラ円を利確。何故しばらく無風だと思ったのかも、どのくらい無風の期間があると思ったのかも思い出せないので。
  - 個人的には、 __ドル円は130円を割って当然__ だと思っている。だからといってそれにベットする勇気があるかというとそうでもないが。
    - TOPIXや日経のインデックスを持っているが、それは処分してもいい気がする。
  - そして、ドルの下落にベットしたい。相手の通貨は、インフレが当面収まらなそうなところが望ましい。
    - 対ドルでの調子はユーロが良いのだが、本当にこれからも？AUDやNZD辺りは利下げが遠そうだから狙いたいが、実績として対ドルで調子が悪い。加ドルも微妙だ。その辺は資源国でもあるから、景気が弱くなると調子も悪くなる。だからといってユーロか？

08/13 火
--------

8:00の目覚ましで9:10起床。

水やりは10:00にペットボトル5本。

出社途中に`ヤホコーヒー`でアイスアメリカーノ1ショット追加をテイクアウト。

__ランチ__ は`松屋`でチャーシューエッグ3枚定食。目玉焼きは3つになってなかった。完全に角煮の味付けだが、日本ではチャーシューの境界は崩壊している。鶏チャーシューさえある。

戻りに`パッセージコーヒー`で __水出しコーヒーをテイクアウト__ 。ニカラグア。

Chromeでのタブ検索はCtrl+Shift+Aだった。Unityのリファレンスを別のウィンドウに集めてタブ検索する便利な環境が戻ってきた。

メッシュを生成したい。Unityでやった方が早いのか、DCCツールで作って貰って、僕がサポートするスクリプトを書くのが早いのか。mayaのライセンス形態に、そういうサポート業務用のレンダリングとかが出来ない安価なプランがあればいいのに。

オフィスの27インチモニタの解像度が4Kだった。27インチで4Kは細かすぎるのではないかと思っていたが、全く問題ない。逆に、自宅の27インチWQHDも問題無いんだけど。

- Unityの愚痴。
  - UnityのVector3にベクトル同士の乗算をする演算子がオーバーロードされてない。`Vector3.Scale(v0, v1)`だってさ。
  - 割り算もない。`Vector3 / float`はあるんだけど、`Vector3 / Vector3`も`float / Vector`もない。せめて逆数を取れれば。
  - `Enumerable.Chunk()`がUnityで使えない。いつものように。
  - これは良かったこと。行列に(0,1,0)を掛けて行を取り出そうとしたら`GetColumn()`が存在した。 GitHub Copilotが教えてくれた。

`Enumerable.Range()`にステップを指定するバージョンが無いのか。引数が個数なのか上限なのか分かりにくいから、`.Select(i => i * step)`の方が明確だということなのかな？

ランチの量が多かったので、今日は間食しないで済んだ。

- 帰宅時に途中まで __ウォーキング__ 。目標は恵比寿。
  - Googleマップで帰宅時のウォーキングに丁度良い目標地点を探していると、目黒線 __白銀高輪駅__ からJR __恵比寿駅__ まで真っすぐ抜ける道がある。恵比寿でクラフトビールを飲んで帰ろう。
  - 暑い。途中でギブアップするかも知れない。
  - 地図を見ている時は、白銀高輪駅近くの`ピーコック`を偵察するつもりだったのが忘れていた。近所に2軒あって、片方にはダイソーが入っているそうだ。
  - 途中の100円ローソンで水を買って飲む。
  - 恵比寿の`Tap＆Tumbler`でボトルの __クラフトビールをテイクアウト__ 。歩きながら飲む。
  - 最近開業した __渋谷駅__ 直結の __サクラステージ__ を見に行く。
    - 思ったよりも敷地面積が小さかった。工事で鉄板で囲われていた範囲が、想像よりも小さかったという話。線路沿いの道が通ったせいもあるのかな？
      - 後でGoogleマップの航空地図で見ると、僕が小さいと思ったのは、線路から遠い側のかなり限られた小さい範囲のようだ。駅側にもっと大きく広がっているらしい？
    - __渋谷ストリーム__ とはもう接続されているのだろうか？ストリーム側から橋を伸ばしているのは見たが。
    - 玉川通り(246)の向こうから地下鉄に乗れるかと思って階段を下がったら、バス停側に出るだけだった。
    - 地下鉄で帰るつもりだったが、丁度渋64のバスがいたので乗り込んだ。一刻も早くエアコンで体を冷やしたかった。

金曜日までの仕事は作業自体は終わらせたがPRを出してなかったので、今日やった。

コレクションを直接`List<T>`などの具体的な型で保持していると`foreach`で余計なアロケーションが起きないが、その`List<T>`を`IReadOnlyList<T>`などで保持するとアロケーションが発生する。これについてボクシングが発生しているからという説明しか見当たらないが、それは完全に正しいが、何も説明してない。イテレーションの状態を保持しなければならないから、新しいオブジェクトが必要になる。その新しいオブジェクトをstructにしてスタックに確保すると、foreachでアロケーションを省く最適化を言語で実現できる。インターフェースで保持しているとstruct版を返すメソッドが見つからないから参照版を呼ぶことになる。その参照版は、単にそのstructをboxして返す。ボクシングが必要だからboxするのではなく、単に工数が低くてシンプルで、そして十分に機能するからだ。なんなら、プールから返すような実装をしたらアロケーションは不要になる。多分マルチスレッドの同期とかで遅くなるけど。

`IReadOnlyList<T>`で保持してforするのと`List<T>`で保持してforeachする、どちらも抵抗がある。

C#のout指定をメソッドの引数のところで書くときのスコープの扱いが嫌い。MicrosoftはC++98の時にも`for`のスコープで同じことをして不評だったのに。
