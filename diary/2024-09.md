- <details open><summary>C#の`event`の解放を少し楽にするために、追加した時に`IDisposable`を返したい。</summary>

  - ```cs
    public class ChainAction<TArg1/*it's good if inherit EventArgs*/> {
      private event Action<object?/*sender*/, TArg1> event = (_, _) > {};
      public void Invoke(object? sender, TArg1 arg1) => event(sender, arg1);
      public IDisposable AddHandler(Action<object?, TArg1> handler) {
        event += handler;
        return new DisposingAction(() => event -= handler);
      }
    }
    ```
  - 色々必要だし、拡張するとスレッドセーフ対応も必要になるけど、ざっくりこういうこと。
  - ほとんどイベントなんだけど、何とかイベントという名前だとC#ネイティブのイベントとの差異が表しにくい。`ChainAction`はまあまあの命名ではないだろうか。
  - `event`の拡張メソッドとして書ければ素直なんだけど、eventはそういうものじゃない。`MulticastDelegate`の拡張メソッドに出来るといいんだけど、実現に必要なメソッドが`protected`なので拡張メソッドではアクセスできない。
