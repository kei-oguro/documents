  - https://kido0617.github.io/unity/2015-01-03-shader-hue-shift/
  - マトリクスを外で生成して渡すようにすると速そう。
  - [僕のやり方のCbCr回転](#1114-rotation-in-YCbCr-space) で、3x3行列を2回、2x2行列を1回乗算するのところを、1つの行列にまとめたもの。
    - `result = float3(dot(mat[0], rgb), dot(mat[1], rgb), dot(mat[2], rgb))` のように内積3回なのを、 `result = matT[0] * r + matT[1] * g + matT[2] * b` のように積和3回にすると、チープなGPUでより速いかも知れない。(matTは転置)
      - <details><summary>このコードをこうする。</summary>

        - コレを、
          ```hlsl
          RESULT.x = (.299*shift.z+.701*VSU+.168*VSW)*RGB.x
              + (.587*shift.z-.587*VSU+.330*VSW)*RGB.y
              + (.114*shift.z-.114*VSU-.497*VSW)*RGB.z;
            
          RESULT.y = (.299*shift.z-.299*VSU-.328*VSW)*RGB.x
              + (.587*shift.z+.413*VSU+.035*VSW)*RGB.y
              + (.114*shift.z-.114*VSU+.292*VSW)*RGB.z;
            
          RESULT.z = (.299*shift.z-.3*VSU+1.25*VSW)*RGB.x
              + (.587*shift.z-.588*VSU-1.05*VSW)*RGB.y
              + (.114*shift.z+.886*VSU-.203*VSW)*RGB.z;
          ```
          こうする。
          ```hlsl
          RESULT.rgb = 
              (float3(.299, .299, .299)*shift.z + float3(.701, -.299, -.3)*VSU + float3(.168, .330, .125)*VSW) * RGB.r +
              (float3(.587, .587, .587)*shift.z + float3(-.587, .413, -.588)*VSU + float3(.330, .035, -1.05)*VSW) * RGB.g +
              (float3(.114, .114, .114)*shift.z + float3(-.114, -.114, .886)*VSU + float3(-.497, .292, 0.203)*VSW) * RGB.b;
          ```
          - で、RGBを掛ける前の値はシェーダの最中に一緒だから、ベクトル3本送ってあげればいい。
        </details>

      - コレは行列による __カラーコレクション__ の、 __HSV変換__ 行列の作り方ということだ。
      - 一つ賢くなった。
  - 最近僕の周りでHSVがホットだけど、探してたのはUnityのUI/Defaultのシェーダの `_Tint` の用途なんだけどね。ボタンでしか使わないのなら使いたいんだけど。
