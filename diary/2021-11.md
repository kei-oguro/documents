    * C#では `public class Class<T> {}` のようなものがあった時に `T` を取り出せない。
      * つまり `var t = new Class<T>.T();` のようなことができない。
      * C++なら `typedef` を使うことで、、 `template <typename T> class Class { typedef Type T; }` のような宣言をすることで `auto t = new Class<T>::Type();` のようにできる。
    * できないと書いたが、外側も partial` じゃないと、外側のクラスの外からpartial inner classを宣言する記法がない。
      * ```cs
        public class Outer { public partial class Inner {} }
        ```
        とあった時に、Outerの外で
        ```cs
        public partial class Outer.Inner {}
        ```
        とは書けないし、
        ```cs
        public partial class Outer { public partial class Inner {} }
        ```
        は元の宣言で「partialがない」とエラーが出るし、
        ```cs
        public class Outer { public partial class Inner {} }
        ```
        元々partialなら宣言できる。
        ではどちらかで「すでに宣言されている」とエラーが出る。

    * 正確に真ん中に出現しない値を作るか、
      ```cs
      uint r = rand();
      float f = (r >> 8) * (float)(1 << 24);    // [0 .. 1<<24)
      return f < .5f ? f : 1.5f - f;
      ```
      * f == 0.5f の時に 1.0f になる。f はピッタリ 1 にならないから、ピッタリ 0.5f が返ることはない。
    * 掛け算で [0 .. 1<<24] の閉区間にしてから変換するか、
      * 本当は32bitレジスタ2つになる掛け算 `(int high, int low) = mul(int a, int b)` が望ましいんだけど。
        ```cs
        ulong ul = rand() * ((1<<24) + 1UL);   // 上位32bitに [0 .. 1<<24] が入っている。
        return (uint)(ul >> 32) * (1.0f / (1 << 24));
        ```
    * 一部の値を却下して剰余で [0 .. 1<<24] の閉区間にしてから変換する。
      ```cs
      uint r = rand();
      while (r >= 255 * ((1<<24) + 1))
        r = rand();
      return (r % ((1<<24)+1)) * (1.0f / (1 << 24));
      ```
      * (1<<24)+1 の余りを得るが、255*((1<<24)+1) 以上の区間が (1<<24)+1 よりも小さな範囲の数字しか表現できないので、そこを却下する。
        * 具体的には255個足りない。255個の区間に1ずつ増やしたのだから当然。
      * M系列では状態変数が完全なゼロになることが無い。
        * 却下する時にそれを考慮したいが、乱数として取り出した値が0の時に、状態変数がゼロとは限らない。
          * というか、内部がゼロなら常にゼロになるので、普通に動いているならゼロじゃない。
        * とはいえ、その状態は必ずゼロになるワケだから、ゼロが却下されると密度が寄り均等になる。
        * ということで却下する範囲にゼロを含めてみよう。
          ```cs
          uint r = rand() - 1;
          while (r >= 255 * ((1<<24) + 1))
            r = rand() - 1;
          return (r % ((1<<24)+1)) * (1.0f / (1 << 24));
          ```
          * 割り算がコスト的に許されるなら、これが最高の精度ではないか。
          * rand() が 0 を返したときには uint.MaxValue になって却下される。
      * 少しだけ精度が上がるが、1.0e-96 だけ。全然いらない。

  * というか、閉区間なんて使う必要が無い。
  * 32bit*32bitの乗算を16bit乗算で行う。
    * (a0 + a1)(b0 + b1) = a0b0 + a0b1 + a1b0 + a1b1
    * a0, b0 は上位16bit。
    * [63 .. 32] a0b0
    * [47 .. 16] a0b1, a1b0
    * [31 .. 0] a1b1 
    * 下位32bitは 
      * middle = (a0b1 & 0xffff) + (a1b0 & 0xffff)
      * carry = (middle + (low >> 16)) >> 16
      * low = a1b1 + (middle << 16)
    * 上位32bitは
      * middle = (a0b1 >> 16) + (a1b0 >> 16)
      * high = a0b0 + middle + carry
  * union trick で作った浮動小数点数が表現可能な範囲を網羅しているか。
    ```cpp
    #include <cstdio>
    #include <cmath>
    #include <vector>

    int main()
    {
        std::vector<unsigned char> appeared((1 << 23) / 8, 0);
        for (int i = 0; i < (1 << 23); ++i)
        {
            const int32_t bits = 0x3f800000 | i;
            const float f = *reinterpret_cast<const float *>(&bits) - 1.0f;
            const float fi = f * (1 << 23);
            if (std::floor(fi) != fi)
            {
                std::printf("%f is not multiple of 2^23.\n", f);
            }
            const int i2 = fi;
            if ((appeared[i2 / 8] & (1 << (i2 & 0x07))) != 0)
            {
                std::printf("%d %d %f is already appeared.\n", i, i2, f);
            }
            appeared[i2 / 8] |= (1 << (i2 & 0x07));
            const float f2 = fi / (1 << 23) + 1.0f;
            const int32_t bits2 = *reinterpret_cast<const int32_t *>(&f2);
            if (bits != bits2)
            {
                std::printf("%f is reformatted by FPU. %d th 0x%08x -> 0x%08x\n", f, i, bits, bits2);
            }
        }
        return 0;
    }
    ```
