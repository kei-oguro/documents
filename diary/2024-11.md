<details open><summary>配列から条件を満たす要素を削除する時、</summary>

```cs
foreach (var @string in strings) {
  if (IsRemoved(@string)) {
    strings.Remove(@string);
  }
}
```
とはできないし、
```cs
for (int i = 0; i < strings.Count; ++i) {}
  if (IsRemoved(strings[i])) {
    strings.RemoveAt(i);
  }
}
```
とすることも出来ない。これは、`RemoveAt()`が要素を切り詰めてしまい、その上に`++i`することでスキップしてしまうからである。
```cs
for (int i = strings.Count - 1; i >= 0; --i) {}
  if (IsRemoved(strings[i])) {
    strings.RemoveAt(i);
  }
}
```
と逆順にすることで、動作する。しかし、別のもっといいことがある。
```cs
int src = 0, dest = 0; // We use them after the loop, so declare here.
for (; src < strings.Count; ++src) {}
  if (IsRemoved(strings[src])) {
    continue; // Skip copy this.
  }
  if (src != dest)
  {
    strings[dest] = strings[src];
  }
  ++dest;
}
string.Remove(dest, src - dest);
```
複数回`RemoveAt()`をすると、何度も要素のコピーが行われる。今例示した方法では、そのコピーが、最終的な位置に一回だけ行われる。

ちなみに、別のコレクションを作って良いのなら、`string.Where(@string => !IsRemoved(@string))`で構わない。インプレースの場合に有効。

ちなみにC#なら`List<T>`にはこれを行う`RemoveAll()`がある。残念なことに`IList<T>`にはない。
</details>
