- __ランチ__ は豆腐の糠漬けと煮豚とカルボナーラ。
  - 豆腐の糠漬けは糠漬け味の豆腐。予想通りの味と食感。もうやらないと思う。
  - カルボナーラはコーヒーを淹れる時に水に漬けておいた乾麺。
    - 100g余ってると思ってたら150gだった。
    - ラードのために挽肉を入れ、少し脂が足りないのでオリーブオイルを入れ、アミノ酸のうまみのために煮豚を刻み、煮凝りを入れ、低温からニンニクを炒める。
    - ニンニクが透き通ってきたら、水戻しパスタを炒め始め、その間に戻し汁と白ワインを合わせたものをレンチン。
    - パスタが透明になったら戻し汁と白ワインを入れて煮詰める。
    - 煮詰まったら火を止めてチーズを入れる。
      - 普通に作るとここでソースの温度が70度程度に下がるので、卵を割り入れて混ぜるとイイ感じのトローリソースになる。粘度が高い時はパスタのゆで汁を加える。
    - 卵を割り入れて撹拌する。
      - 今日は、パスタがすでに入っていたせいで、パンの温度が十分に下がらず、卵が固まってしまった。
      - それと、150gでSサイズの卵1つは少ない。もう1つ追加すべきだった。
    - 黒コショウを掛けて出来上がり。
    - 十分に美味しかったが、ソースの粘度が無いので寂しい。
      - そして、麺が少し硬い。レンチンしたのを入れるべきかも知れない。

- <details><summary>プログラムの話。</summary>

  - 二重初期化のバグを検出できる仕組みを作るのは簡単ではない。
  - 初期化済みでなければ呼んではいけないメソッドについて、マークを付けることを考える。
  - まずは、マークが付いていたら、初期化済みかどうかチェックするコードを挿入するツールが考えられる。
  - 次に、変数に対してマークを付けることを考える。
    - cv modifierのように、 `initialized Object * obj = nullptr;` と修飾子を付けて宣言できるようにする。
    - `initialized Object Object::Initialize() { return this; }` のような呼び出しで修飾されたインスタンスを返し、コンパイラでチェックするように出来るのではないか。
  - この仕組みを、初期化に限らずにより一般的な仕様にするのが望ましいと思う。
  - 例えば、「ある関数を呼んでいい」ことを表すようなことが出来ないか。
    - スレッド安全性のために、関数をグループ化して、対応する関数しか呼べない、というようなことができると、ロックが必要かどうかとかのチェックにいくらか活かせると思う。
    - 例えばスレッドで producer/consumer パターンを実装する時、「producerからしか呼んではいけない/producerからだけ呼んではいけない」修飾子によってチェックするようなことは役に立つのではないか。
    - そして、「ある関数から呼び出されている関数はproducerである」というような定義をして、静的に解析できると、グループ分けのミスも減らせると思う。
  - 言語を自作するようなやる気が起きるとは思えないけど。
  </details>
